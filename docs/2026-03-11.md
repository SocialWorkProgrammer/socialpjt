## 공공데이터 API 활용 계획
1. **사용 API**: 공공데이터포털에 등록된 3종 서비스
   - 중앙서비스 API
   - 지자체중앙서비스 API
   - 복지서비스정보 API
2. **인증/환경 변수**
   - `.env`에 API 키를 저장(`PUBLIC_API_KEY` 또는 추후 `SOCIAL_SERVICE_API_KEY`).
   - `services` 앱에서 관리 명령(`fetch_social_services`)이 해당 키를 헤더/쿼리에 포함해 호출.
3. **데이터 수집 전략**
   - 각 API 별로 페이지네이션·필수 파라미터 확인 후 하루 1회 수집(뉴스와 동일한 재시도 정책, 실패 시 상태 기록 및 팝업 “금일 사회서비스 정보를 확인할 수 없습니다” 출력).
   - 응답 필드 매핑: 서비스 ID, 명칭, 요약/본문, 제공 기관, 신청 방법, 대상자 조건, 지역, 신청 URL 등을 `SocialService` 모델에 저장.
   - `external_id + source` 기준 `update_or_create`로 중복 방지, `ServiceFetchStatus` 모델로 성공/실패 로그.
4. **게시판 기능**
   - 댓글 기능은 제외.
   - 검색(키워드, 카테고리, 지역, 출처) + Pagination 제공, UI는 커뮤니티/뉴스 스타일 재사용.
